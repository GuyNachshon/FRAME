## Task 1 — Fix four known bugs before anything else

### 1a. Fix checkpoint tracking in `predictor/train.py`

**Current (wrong):** saves best checkpoint when `pred_loss` is lowest.
**Correct:** saves best checkpoint when `action_sensitivity` is highest.

Action sensitivity is the metric that determines demo quality — proved by
sweeping all 40 checkpoints from run 1. The checkpoint with lowest pred_loss
(step 60k) ranked 22nd out of 40 by action sensitivity.

Implementation:
- At every `eval_every` interval, compute action sensitivity on a held-out
  batch of 50 samples using the same logic as `eval/action_sensitivity.py`
- Track EMA of action sensitivity: `ema_sens = 0.9 * ema_sens + 0.1 * sens`
- Save `predictor_best.pt` when `ema_sens > best_ema_sens`
- Log `eval/action_sensitivity` to wandb at every eval interval

### 1b. Fix `inv_acc` at checkpoint time in `predictor/train.py`

**Current:** evaluates inverse dynamics accuracy on 1 batch of 4 sequences.
**Correct:** evaluate over 50 batches minimum, report the mean.

The current implementation produces 0.0, 0.25, 0.5, 0.75, or 1.0 — pure
noise from a single batch on a 72-class problem. It's meaningless.

### 1c. Fix display fps in `inference/display.py`

**Current:** pygame window is 512×512, using `pygame.transform.scale` from
128×128. This takes 229ms per frame on macOS = 4fps.

**Fix:** change window to 256×256 and remove or reduce the upscale:
```python
screen = pygame.display.set_mode((256, 256))
# If scaling is needed, use pygame.SCALED flag instead
```

Target: ≥15fps on macOS MPS.

### 1d. Verify keyboard capture in `inference/loop.py`

Add a debug print to confirm actions reach the model:

## Task 2 — New dataset: ViZDoom Deathmatch PPO XLrg

**Source:** `P-H-B-D-a16z/ViZDoom-Deathmatch-PPO-XLrg` on HuggingFace
**Size:** ~64k rows, 49.5GB, ~640k frames
**Content:** PPO-trained agent playing ViZDoom deathmatch over 15 episodes
**Format:** parquet files with columns: `episode_id`, `step_id`, `health`,
`actions` (list of 10 ints), `images` (list of 10 base64 PNG strings)

### Why this dataset

- PPO agent actually navigates — runs, strafes, engages enemies
- Deathmatch map has full spatial diversity vs single-scenario maps
- 640k frames vs our 200k = 3× more diverse training data
- Pre-labeled actions, no collection needed
- Expected frame diff: 20-35 (vs 5-7 from our collected data)

### 2a. Write a dataset loader `data/hf_deathmatch/loader.py`

Requirements:
- Download using HuggingFace `datasets` library
- Decode base64 PNG images to numpy uint8 arrays
- Resize frames to 128×128 if not already that resolution
- Map integer actions (0-17 range) to our 8-dim one-hot format

**Action mapping** — ViZDoom deathmatch action integers to our 8-key vocab:
```python
# ViZDoom deathmatch has a larger action space than our 8 keys
# Map to our format: [forward, back, left, right, turn-left, turn-right, shoot, noop]
# Actions involving forward movement → forward
# Actions involving backward → back
# Actions involving left strafe → left
# Actions involving right strafe → right  
# Actions involving left turn → turn-left
# Actions involving right turn → turn-right
# Actions involving attack → shoot
# All others → noop
# Note: multiple actions can be active simultaneously in deathmatch
# For our one-hot format, use the primary action (highest priority)
```

Inspect the actual action integers in the dataset first:
```python

```

- Save output as HDF5 with same schema as existing datasets:
  - `frames`: (N, 128, 128, 3) uint8
  - `actions`: (N, 8) float32 one-hot
  - `episode_ids`: (N,) int32

## Task 3 — Training infrastructure improvements

### 3a. Add BF16 mixed precision to `predictor/train.py`

Use PyTorch AMP with BF16. RTX 4090 has native BF16 tensor cores.
Expected speedup: 1.5-2×.

```python
from torch.cuda.amp import GradScaler
from torch import autocast

scaler = GradScaler()

# In training loop:
with autocast(device_type='cuda', dtype=torch.bfloat16):
    logits, info = predictor(tokens, actions, scene_state, gru_state)
    loss = compute_loss(logits, targets, inv_logits, inv_targets)

scaler.scale(loss).backward()
scaler.clip_grad_norm_(predictor.parameters(), config['training']['grad_clip'])
scaler.step(optimizer)
scaler.update()
```

Also add BF16 to `tokenizer/train.py` if not already present.

### 3b. Add Flash Attention to `predictor/transformer.py`

Replace standard attention with Flash Attention 2:

```bash
pip install flash-attn --no-build-isolation
```

```python
# In transformer.py, in the attention forward:
try:
    from flash_attn import flash_attn_func
    # Use flash attention when available
    USE_FLASH = True
except ImportError:
    USE_FLASH = False
```

Make it a graceful fallback — if flash-attn is not installed, use standard
attention. This keeps the codebase portable.

Expected speedup: 1.3-1.5× on attention layers. Also enables extending
context window from 8 to 16 frames at same memory cost in future.
